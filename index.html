Wacky Fill-Ins

The Basic Idea

Basically, you have an incomplete story with blanks, and you can fill in those blanks to create a finished story that might not make much sense narratively, but will hopefully be unexpectedly hillarious. See README.md for a more de­tailed explanation of the concept.

Try It Out

I have included three samples to demonstrate the concept:

  1. Running Into an Old Acquaintance — A famous woman runs into an old friend
  2. Interstellar Hijinks — A transmission from a nebula could be trouble for Earth's moon
  3. Artist Statement — Artwork by John K. Ford questions the boundaries of elusiveness

How to Write Your Own Wacky Fill-Ins

You don't have to know JavaScript to write your own Wacky Fill-Ins, but it really helps to know HTML, and, to a lesser extent, CSS. You can use the provided template to get started.

My friend Cole suggested a creation wizard that would enable some­one to write a Wacky Fill-In story without needing to know any HTML markup at all. One would only need a rudimentary and self-ex­plan­a­to­ry markdown.

Cole did a lot of the heavy lifting for the wizard. Now I just need to sit down and wire a couple of things to make it work. Should be easy but not trivial.

So, for the time being, the best way to make a Wacky Fill-In story is to du­pli­cate the HTML template and edit it in your favorite plain text editor.

The story goes inside a div tag with id="libbable". However, you might find it helpful to use id="libbableDraft" when you're drafting your story. Inside the div, you will use p tags to make paragraphs.

To indicate a blank space that needs to be filled in, use a span tag with class="fillable". Be sure to also give the span an id if you want to reuse the fill-in later in the story. In between the opening and closing of the span tag, write the de­scrip­tion of what needs to be filled in, e.g., "verb", "communications device", "dog breed".

To repeat a previous fill-in, use a span tag with class="repeatFill". Write the id of the field you want to repeat in between the opening and closing for the span tag that is to have the repeated fill-in. For example:

<p>Little <span id="firstName" class="fillable">first name</span> the <span id="animalSpecies" class="fillable">animal species, cap­i­tal­ized</span> was walking down <span class="fillable">street</span>. Little <span class="repeatFill">firstName</span> the <span class="repeatFill">animalSpecies</span> enjoyed <span class="fillable">activity, gerund</span> from time to time.</p>

If you preview that in your Web browser (with the libbableDraft styles), it should look something like this:

Little first name the animal species, cap­i­tal­ized was walking down street. Little firstName the an­i­mal­Species enjoyed activity, gerund from time to time.

Once you're satisfied with your story and its blanks, it's time to test it out. In the head section, find these two lines:

<!-- REACTIVATE WHEN READY: script defer src="libbable.js"></script -->
<!-- REACTIVATE WHEN READY: script defer src="index.js"></script -->

Edit them so they look like this:

<script defer src="libbable.js"></script>
<script defer src="index.js"></script>

While you're at it, also edit the meta tags to include a short de­scrip­tion of your story, and your name in the author field.

Also be sure the div has id="libbable", not id="libbableDraft". This actually hides the story, but it also enables the script to create the form with the appropriate fields to correspond to the blanks.

In the following example, the fields are read-only. Go to one of the samples linked above to see a form that you can fill in the fields of.

first name
animal species, cap­i­tal­ized
street
activity, gerund

Once all the fields are filled in, the button to transcribe the fields into the story should become enabled. Click it and the story should ap­pear with the blanks filled in. For example:

Little Jennyfirst name the Jaguaranimal species, cap­i­tal­ized was walking down Woodward Avenuestreet. Little Jenny the Jaguar enjoyed eating tomatoesactivity, gerund from time to time.

If you really like how a filled-in story turns out, you can print it out. I have provided a style sheet for the purpose.

Authorship

Any Wacky Fill-Ins stories with blanks that you create are of course your own work of authorship.

The authorship of a filled-in story that was filled in by a group of people, well, I suppose that's group authorship, but I doubt anyone would care too much about it.

I created this implementation, including the first draft of the Ja­va­Script. Cole Cooper Wyman rewrote the JavaScript for elegance, ef­fi­cien­cy and to comply with modern JavaScript best practices.

I also wrote the three samples, as well as the HTML template. However, the template has a placeholder for the author's name in the metadata.
